<h1 class="ghq-card-content__large-heading" data-ghq-card-content-type="LARGE_HEADING">
 Cheque Book Issuance &amp; Reordering Matrix
</h1>
<p class="ghq-card-content__paragraph" data-ghq-card-content-type="paragraph">
 <strong class="ghq-card-content__bold" data-ghq-card-content-type="BOLD">
  Classification key points:
 </strong>
</p>
<ul class="ghq-card-content__bulleted-list" data-ghq-card-content-type="BULLETED_LIST">
 <li class="ghq-card-content__bulleted-list-item" data-ghq-card-content-type="BULLETED_LIST_ITEM">
  <strong class="ghq-card-content__bold" data-ghq-card-content-type="BOLD">
   AECB Data:
  </strong>
  New to Banking system classification (Months in the system) → &lt;12 month (&lt;365 days)
 </li>
 <li class="ghq-card-content__bulleted-list-item" data-ghq-card-content-type="BULLETED_LIST_ITEM">
  <strong class="ghq-card-content__bold" data-ghq-card-content-type="BOLD">
   Internal (Wio) Data:
  </strong>
  Time period of Wio data to calculate slabs →  0; 1-3 → 12 month (365 days)
 </li>
 <li class="ghq-card-content__bulleted-list-item" data-ghq-card-content-type="BULLETED_LIST_ITEM">
  Only 10 leaves cheque book will be allowed for Retail.
 </li>
</ul>
<p class="ghq-card-content__paragraph ghq-is-empty" data-ghq-card-content-type="paragraph">
</p>
<p class="ghq-card-content__paragraph" data-ghq-card-content-type="paragraph">
 <strong class="ghq-card-content__bold" data-ghq-card-content-type="BOLD">
  <span class="ghq-card-content__text-color" data-ghq-card-content-type="TEXT_COLOR" style="color:#9013fe">
   Below is the flow for checking eligibility of New to Bank [NTB] customers (First time cheque book order):
  </span>
 </strong>
</p>
<ol class="ghq-card-content__numbered-list" data-ghq-card-content-type="NUMBERED_LIST" start="1">
 <li class="ghq-card-content__numbered-list-item" data-ghq-card-content-type="NUMBERED_LIST_ITEM">
  Once a user clicks on “Order Cheque Book”,  we check the cache to see if we have AECB data. If AECB data indicates that the user is not eligible for to receive cheque book (Number of bounced cheque &gt;=4), then the user journey ends there and we notify that the customer is not eligible to receive cheque book.
 </li>
 <li class="ghq-card-content__numbered-list-item" data-ghq-card-content-type="NUMBERED_LIST_ITEM">
  On the “Order confirmation screen”, the user would not have the option to select the type of cheque book. By default, the user would only be provided a 10 leaves cheque book.
 </li>
 <li class="ghq-card-content__numbered-list-item" data-ghq-card-content-type="NUMBERED_LIST_ITEM">
  Once the user completes the confirmation, we will initiate a call to AECB to cheque for number of bounced cheques. Almost all the users should be eligible, but, if a user is not eligible (number of bounced cheques &gt;=4), then we notify the user (via Email etc.) that they are not eligible to receive cheque books.
 </li>
</ol>
